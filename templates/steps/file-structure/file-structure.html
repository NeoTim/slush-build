Understanding your File Structure

This Generator is provided as a way for you to learn how to properly structure your App Architecture, by following John Papas Angularjs Style Guide, and Google Angular Style Guide

Getting Started

Take a look at the current file structure

  • app
    • Contains all the Javascript Logic for the Angular Application
    • Also contains all the HTML views we will use thoughout our application.
  • bower_components
    • Contains all the bower components downloaded, and listed in the bower.json file.
  • styles
    • Contains all the app css styles
  • .bowerrc
    • This file tells bower where to place the install dependencies
    • For example: if you don't like using the word bower_components then change the name in .bowerrc
  • .editorconfig
    • Tells the current editor you are using, to use specific configurations
  • index.html
    • I am assuming you know what this file does...
    • Here is the page the contains the entire single page application.
  • package.json
    • Similar to bower.json, Howerver, it is for files install view npm
